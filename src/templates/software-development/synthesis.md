## Synthesis Instructions

Produce a comprehensive technical report with the following sections:

### Executive Summary

Summarize what was built, the key design decisions made, and the overall status of the deliverable. Highlight any significant risks or open items. Keep this to 3-5 sentences.

### Architecture

Summarize the architect's design including:
- Major components and their responsibilities
- Data model and key entities
- Interface contracts and communication patterns
- Non-functional design decisions (caching, scalability, security)

Note any deviations between the original design and the final implementation.

### Implementation

Describe what was built including:
- Code organization and module structure
- Key algorithms or business logic implemented
- Error handling and resilience patterns applied
- Configuration and deployment considerations

Flag any areas where the implementer noted design gaps or made independent decisions.

### Test Coverage

Report on testing completeness:
- Summary of test categories and case counts
- Coverage of happy paths, error scenarios, and edge cases
- Identified risk areas and gaps in coverage
- Recommendations for additional testing

### Documentation

Assess documentation completeness:
- Which deliverables were produced (API reference, architecture overview, guides, examples)
- Quality and accuracy of the documentation
- Any items flagged as needing clarification

### Recommendations

Provide actionable next steps:
- Critical issues that must be addressed before release
- Improvements that should be scheduled for follow-up
- Technical debt identified during the process
- Suggested enhancements based on the team's findings
